# UE资产分析功能说明

## 概述
UE资产分析是本工具的核心功能，允许您拖拽或选择UE资产文件（.uasset、.umap等）进行深度分析。

## 功能详解

### 1. 识别UE版本
通过解析资产文件的头部数据，识别该资产属于哪个虚幻引擎版本。
- 支持从 UE 4.0 到 UE 5.2+ 的版本识别
- 显示版本号和版本范围

### 2. 分析文件信息
- **文件大小**: 显示资产文件的实际大小（KB为单位）
- **资产类型**: 识别资产类型（蓝图、关卡、网格、材质等）
- **文件扩展名**: 显示原始文件扩展名

### 3. 列出关联文件
识别与该资产相关的其他文件：
- `.ubulk` 文件：资产的数据块文件
- `.uexp` 文件：资产的导出数据
- 其他相同基名但不同扩展名的文件

### 4. 解析导入/引用
提取资产文件中引用的其他资产路径：
- 以 `/Game/` 开头的资产路径
- 以 `/Engine/` 开头的引擎资产
- 其他脚本类路径

### 5. 提取UE内路径
尝试从资产文件中提取其在虚幻引擎项目中的完整路径。

## 使用步骤

### 方法1：拖拽文件
1. 打开工具并切换到"资产分析"标签页
2. 从文件管理器中拖拽 .uasset 或 .umap 文件
3. 拖拽到蓝色虚线框区域
4. 自动分析并显示结果

### 方法2：浏览选择
1. 点击"浏览"按钮
2. 在文件对话框中选择UE资产文件
3. 点击"分析"按钮
4. 等待分析完成并查看结果

## 支持的文件格式

- `.uasset` - 虚幻资产主文件
- `.umap` - 虚幻地图/关卡文件
- `.uexp` - 资产导出数据
- `.ubulk` - 资产数据块

## 技术细节

### 文件格式识别
工具通过检查文件魔数（0x9E2A83C1）来识别有效的UE资产文件。

### 版本识别
从文件头的第5-8字节读取版本信息，映射到相应的UE版本范围。

### 引用提取
通过搜索文件内容中的路径前缀（如 `/Game/`、`/Engine/`）来提取资产引用。

## 注意事项

1. **文件完整性**: 确保拖拽的是完整的资产文件，不是损坏的文件
2. **引擎版本**: 不同UE版本的资产文件格式可能略有差异，本工具尽力兼容
3. **引用提取**: 对于某些加密或特殊格式的资产，可能无法完全提取引用信息
4. **性能**: 对于非常大的资产文件（>1GB），分析可能需要较长时间
