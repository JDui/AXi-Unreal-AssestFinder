# UE资产路径查找与分析工具 - 完整功能说明

## 工具概述

这是一个专为虚幻引擎(UE)开发者设计的强大工具，集资产文件分析和路径转换于一身。

## 核心功能

### 1️⃣ UE资产文件分析（新功能）

#### 功能特点
- **拖拽分析**: 直接拖拽 .uasset 或 .umap 文件进行实时分析
- **版本识别**: 自动识别资产属于的虚幻引擎版本（UE 4.0 - UE 5.2+）
- **文件信息**: 显示文件大小、资产类型等详细信息
- **关联文件**: 列出所有相关文件（.uexp、.ubulk等）
- **引用解析**: 提取资产引用的其他资产路径
- **路径提取**: 尝试从资产文件中提取其在UE项目中的完整路径

#### 分析结果包含
```
📄 基本信息 → 文件名、大小、类型
🎮 引擎信息 → UE版本、版本号、魔数
📁 关联文件 → .uexp、.ubulk等相关文件列表
🔗 导入引用 → /Game/ 路径下的资产引用
📍 UE内路径 → 资产在项目中的完整路径
```

### 2️⃣ 文件系统路径转换

#### 功能特点
- **单文件转换**: 输入任意文件路径，自动转换为UE Content格式
- **批量搜索**: 扫描整个目录，识别所有UE资产
- **智能分类**: 根据文件类型自动建议存放目录
- **树形展示**: 按资产类型分组展示结果
- **快速复制**: 一键复制转换后的UE路径

#### 支持的资产类型
- 贴图（.png, .tga, .jpg等）
- 网格（.fbx, .usmf）
- 蓝图（包含"blueprint"或"bp_"的文件）
- 材质（包含"material"的文件）
- 动画（包含"animation"或"anim"的文件）
- 音效（.wav, .mp3）
- 粒子（.umpc）
- 关卡（.umap）
- 其他资产

## 工作流示例

### 场景1: 分析现有UE资产
```
1. 启动工具 → 选择"资产分析"标签页
2. 拖拽 MyCharacter.uasset 文件
3. 查看分析结果：
   ✓ 识别为 UE 4.27 资产
   ✓ 显示关联的 .uexp 和 .ubulk 文件
   ✓ 列出引用的资产路径
```

### 场景2: 转换新资产路径
```
1. 启动工具 → 选择"路径转换"标签页
2. 点击"浏览"选择 D:\Projects\Assets\character_model.fbx
3. 点击"转换"
4. 查看结果：
   📍 推荐目录: Characters
   📍 UE路径: /Game/Characters/character_model.fbx
5. 点击"复制路径"到剪贴板
```

### 场景3: 批量处理目录
```
1. 启动工具 → 选择"路径转换"标签页
2. 点击"浏览"选择 D:\UE_Assets 目录
3. 点击"搜索"
4. 树形视图按类型显示：
   ├─ 【网格文件】(3个)
   │  ├─ character.fbx → /Game/Characters/character.fbx
   │  ├─ weapon.fbx → /Game/Characters/weapon.fbx
   │  └─ ...
   ├─ 【贴图文件】(5个)
   │  └─ ...
   └─ ...
```

## 技术细节

### UE资产文件格式
```
字节 0-3:    魔数 (0x9E2A83C1)
字节 4-7:    版本号
字节 8-11:   自定义版本
字节 12+:    资产数据 + 嵌入路径引用
```

### 版本映射
- UE 4.0-4.3  → 版本 1
- UE 4.4-4.6  → 版本 2
- UE 4.7-4.8  → 版本 3
- ...
- UE 5.0-5.1  → 版本 13
- UE 5.2+     → 版本 14

### 路径转换规则
```
输入: D:\Assets\Models\character.fbx
↓
文件名分析 → character
文件类型分析 → .fbx (网格)
↓
推荐目录 → Characters
↓
输出: /Game/Characters/character.fbx
```

## 故障排除

### 拖拽不工作
- 检查是否安装了 tkinterdnd2
- 尝试使用"浏览"按钮替代

### 无法识别资产
- 检查文件是否是有效的UE资产文件
- 确认文件完整性（不是损坏的文件）

### 版本识别错误
- 某些特殊的UE版本可能无法精确识别
- 使用文件的版本号自行判断
