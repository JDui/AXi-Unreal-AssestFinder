## 技术实现细节

### 1. UE资产解析 (ue_parser.py)

#### 文件格式识别
```python
魔数检测: 0x9E2A83C1
版本识别: 从文件头第5-8字节
路径提取: 搜索 /Game/, /Engine/ 前缀
```

#### 支持版本范围
- UE 4.0 - 4.3 (版本1)
- UE 4.4 - 4.6 (版本2)
- ... (共14个版本范围)
- UE 5.0 - 5.1 (版本13)
- UE 5.2+ (版本14)

#### 资产类型识别
- 通过文件扩展名
- 通过文件内容特征
- 通过文件名规则

### 2. 路径转换 (asset_finder.py)

#### 转换规则
```
输入: D:\Assets\Models\character.fbx
      ↓ 分析文件名: character
      ↓ 识别类型: .fbx → 网格类型
      ↓ 查询规则: 包含"character" → Characters目录
      ↓ 生成路径: /Game/Characters/character.fbx
输出: UE Content路径
```

#### 目录映射
```python
Characters     ← 角色和人物模型
Meshes         ← 网格模型
Materials      ← 材质
Textures       ← 贴图
Effects        ← 特效
Animations     ← 动画
Sounds         ← 音频
Blueprints     ← 蓝图
Maps           ← 地图/关卡
Widgets        ← UI控件
...
```

UEAssetFinderGUI (主类)

#### 事件处理
```python
拖拽事件    → on_drop_asset() → analyze_asset()

```

## 已知限制

1. **文件大小限制**: 对于超大文件（>2GB）可能出现性能下降
2. **加密资产**: 某些加密或特殊格式资产无法解析引用
3. **自定义引擎**: 基于UE源码修改的自定义版本可能无法识别
4. **路径提取**: 某些资产的UE内路径可能无法完全提取
